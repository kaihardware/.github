# Contributing to KAI

Thank you for helping refine the tools people love.

## Before starting

Search existing issues. For non-trivial changes, open the relevant Problem, Research, Feature, or Design Discussion form before investing substantial work. Do not disclose vulnerabilities publicly; follow SECURITY.md.

## Contribution path

1. Agree on the problem and acceptance evidence.
2. Create a focused branch and small commits.
3. Update requirements, decisions, and verification evidence with the change.
4. Open a pull request using the template.
5. Address review and keep human accountability explicit for AI-assisted work.

## AI-assisted contributions

AI tools are welcome. The contributor remains responsible for correctness, provenance, licensing, confidentiality, and testing. Never send secrets, personal data, supplier-confidential files, or unreleased CAD to an unapproved service. State material AI assistance in the pull request and verify generated content.

## Files and provenance

Do not commit generated exports without their source or regeneration notes. Use Git LFS where configured. Confirm third-party licenses and retain attribution. A Signed-off-by line may be required later if KAI adopts a Developer Certificate of Origin.
